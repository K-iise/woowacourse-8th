# 🎰 로또 프로그램

## ⚙️ 기능 목록 정리

### **1. 입력**

- [x] 사용자로부터 **로또 구입 금액**을 입력받는다.
    - 구입 금액은 **1,000원 단위**로 입력받는다.
    - 1,000원으로 나누어 떨어지지 않는 경우 → `IllegalStateException` 발생
    - 0원 이하의 금액을 입력 받은 경우 -> `IllegalStateException` 발생

- [x] 사용자로부터 **당첨 번호 6개**를 입력받는다.
    - 번호는 쉼표(`,`)를 기준으로 구분한다.
    - 각 번호는 **1 ~ 45 범위**에 속해야 한다.
        - 범위를 벗어난 숫자가 포함된 경우 → `IllegalStateException` 발생
    - 당첨 번호 6개는 **중복될 수 없다.**
        - 중복된 번호가 있는 경우 → `IllegalStateException` 발생

- [x] 사용자로부터 **보너스 번호**를 입력받는다.
    - 보너스 번호는 **1 ~ 45 범위**여야 한다.
        - 범위를 벗어난 숫자가 포함된 경우 → `IllegalStateException` 발생
    - 보너스 번호는 **당첨 번호 6개와 중복될 수 없다.**
        - 당첨 번호와 중복될 경우 → `IllegalStateException` 발생

---

### **2. 로또 발행**

- [x] 로또 1장의 구입 금액은 **1,000원**으로 설정한다.
- [x] 구입 금액에 따라 **발행할 로또 개수**를 계산한다.
- [x] 각 로또는 **1 ~ 45 사이의 중복되지 않은 6개의 숫자**로 구성된다.
- [x] 발행된 각 로또의 번호는 **오름차순으로 정렬**한다.

---

### **3. 당첨 결과 계산**

#### 3-1 당첨 비교
- [x] 사용자가 구입한 각 로또를 **당첨 번호와 비교**한다.
- [x] 각 로또별로 **일치한 번호 개수**를 계산한다.
- [x] **보너스 번호 일치 여부**를 확인한다.
- [x] 일치 개수와 보너스 번호 여부를 기반으로 **등수를 판정**한다.

| 등수 | 조건 | 상금 |
|------|------|------|
| 1등 | 6개 번호 일치 | 2,000,000,000원 |
| 2등 | 5개 번호 + 보너스 번호 일치 | 30,000,000원 |
| 3등 | 5개 번호 일치 | 1,500,000원 |
| 4등 | 4개 번호 일치 | 50,000원 |
| 5등 | 3개 번호 일치 | 5,000원 |

#### 3-2 결과 집계
- [x] 각 등수별로 **당첨된 개수**를 카운트한다.
- [x] **당첨 결과를 저장**한다. 

#### 3-3 수익률 계산
- [x] 모든 당첨 결과의 **총 상금**을 합산한다.
- [x] **총 구입 금액 대비 수익률**을 계산한다.

---

### **4. 출력**

- [x] **발행한 로또 수량 및 번호**를 출력한다.
    - 각 로또 번호는 **오름차순 정렬된 상태**로 표시한다.
- [x] **등수별 당첨 내역**을 출력한다.
- [ ] **총 수익률**을 출력한다.
    - 수익률은 **소수점 둘째 자리에서 반올림하여 출력**한다.

## 🧱 아키텍처 설계

- **Controller**  
  View로부터 입력을 받아 Domain과 Service를 연결하며, 전체 로또 프로그램 흐름을 제어합니다.
    - `LottoController` : 로또 구매, 당첨 번호 확인, 결과 출력까지의 흐름 관리

- **View**  
  사용자 입력과 출력 담당
    - `InputView` : 구매 금액, 수동 번호 입력 등 처리
    - `OutputView` : 로또 결과 및 당첨 내역 출력

- **Service**  
  입력값 검증, 로또 생성 등 핵심 비즈니스 로직 처리
    - `LottoService` : 로또 구매, 당첨 확인, 통계 계산
    - `LottoGenerator` : 로또 번호 생성 인터페이스
    - `RandomLottoGenerator` : 랜덤 로또 번호 생성 구현
    - `Parser` : 입력 문자열을 도메인 객체로 변환

- **Domain**  
  핵심 비즈니스 객체와 상태 관리
    - `Lotto` : 단일 로또 번호 집합
    - `LottoPurchase` : 사용자가 구매한 로또 정보
    - `Lottos` : 여러 로또를 모아 관리
    - `WinningLotto` : 당첨 번호 및 보너스 번호
    - `LottoResult` : 당첨 통계와 총 당첨금 관리
    - `Rank` : 당첨 등수 및 상금 정보

- **Exception**  
  프로그램 예외 및 오류 메시지 관리
    - `ErrorMessage` : 예외 상황별 메시지 정의

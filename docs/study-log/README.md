## 학습법 v3

- 파악된 키워드들이 기존에 내가 알던 기술과 어떤 연관이 있는지(차이점/공통점) 문장으로 연결해 본다.
### 2. 문제(목적)을 해결하기 위한 최소한의 개념만 학습한다.

- 구현 과정에서 필요한 개념에 대해서 선택적으로 깊게 학습한다.

- 이 과정에서 AI를 적극적으로 이용한다.

- 모든 내용을 깊게 이해하려고 하기 보단, 현재 문제에 대해서 해결할 수 있는 수준까지 학습한다.

- 정리한 내용을 바탕으로 크루들과 공유하고, 질문과 피드백을 통해 이해를 보완한다.

- 설명이 어렵거나 막히는 부분은 다시 찾아보며 보완한다.
### 4. AI 페어 리뷰를 통해 논리를 검증하고 '지식의 저주'를 해소한다.  (추가)

- 크루들에게 공유하기 전, AI에게 '시니어 개발자' 페르소나를 부여하고 내 정리에 대해 압박 면접(논리적 허점 지적)을 요청한다.

## 학습법 v3-2 (비유 보단 결핍으로)
### 0. 학습이 필요한 이유와 목적을 명확하게 설정한다.
- 이번 학습에서 해결하고 싶은 문제 또는 구현 목표를 한 문장으로 정의한다.

### 1. 키워드를 이용해서 새로운 개념에 대한 흐름을 파악한다.
- 공식 문서, 블로그, AI를 활용해 10~15분 내로 핵심 키워드 2~3개만 추린다.
- 모든 내용을 이해하려 하기보다, “어떤 문제를 해결하는 기술인지” 수준까지만 파악한다.

### 2. 코드 실험을 통한 결핍 체감 (Action & Fail)
- 최소 기능 예제 코드를 실행해보고, 의도적으로 에러를 발생시켜본다.
- 이 기술이 없을 때 발생하는 불편함이나 반복 작업을 직접 경험한다.
- 단순히 사용 방법을 익히는 것이 아니라, “왜 이게 필요한지”를 몸으로 느끼는 것을 목표로 한다.

### 3. 문제(목적)을 해결하기 위한 최소한의 개념만 학습한다.
- 구현 중 막히는 지점에 한해서 관련 개념을 선택적으로 학습한다.
- AI와 공식 문서를 활용하되, “지금 문제 해결에 필요한 수준까지만” 이해한다.

### 4. ‘결핍 → 해결’ 구조로 나만의 언어로 정리한다

- “이 기술이 없었을 때의 문제 → 어떻게 해결되는지” 순서로 정리한다.
- 정의보다 문제 상황과 코드 중심으로 설명한다.
- 정리한 내용을 바탕으로 크루들과 공유하고, 질문과 피드백을 통해 이해를 보완한다.
- 스스로 설명이 막히는 부분은 다시 찾아보며 보완한다.

### 5. AI 페어 리뷰로 논리를 검증한다

- 크루들에게 공유하기 전, AI에게 시니어 개발자 역할을 부여하고, 내 정리에 대한 반박과 질문을 요청한다.
- 답변하는 과정에서 이해가 부족한 부분을 식별하고 보완한다.
